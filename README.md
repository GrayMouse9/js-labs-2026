## Цель и задачи лабораторной работы

**Цель:** Знакомство с платформой Node.js, пакетным менеджером npm и написание простого одностраничного веб-приложения (SPA) на чистом JavaScript.

**Задачи, реализованные в ходе работы:**
1. Подготовка рабочего окружения, инициализация проекта через `npm init` и настройка `.gitignore`.
2. Подключение библиотеки компонентов Bootstrap 5 через npm.
3. Разделение монолитного кода на модульную архитектуру: выделение независимых страниц (`pages`) и переиспользуемых UI-элементов (`components`).
4. Реализация динамического рендеринга HTML-контента через JavaScript (работа с DOM-деревом без прямых HTML-файлов для каждой страницы).
5. Выполнение индивидуального задания (Вариант 3): адаптация контента под заданную тему и внедрение обязательного Bootstrap-компонента.

## Индивидуальная тематика и стилистика

В рамках индивидуального задания (Вариант 3) базовый код из методички был полностью переработан:

* **Тематика:** Интерфейс баллистического центра по расчету космических траекторий перелета к Луне.
* **Терминология:** Согласно строгим требованиям к защите, абстрактные "Продукты" (Products) были переименованы в "Услуги" (Services).
* **Обязательный компонент Bootstrap:** Значки (Badges). Использованы для визуальной маркировки особенностей траекторий (например, "Быстрый", "Экономичный").
* **Верстка:** Карточки услуг выстроены плиткой с выравниванием по левому краю (`justify-content-start`).

## Теоретическая база применяемых технологий

При разработке приложения были применены следующие концепции и API:
1. **DOM API:** Использование методов `getElementById` для поиска корневых контейнеров и `insertAdjacentHTML` для безопасной вставки сгенерированной HTML-разметки на страницу.
2. **ES6 Модули:** Применение директив `import` и `export` для связывания разрозненных JS-файлов в единое приложение.
3. **Событийно-ориентированная модель:** Подписка на действия пользователя с помощью слушателей `addEventListener` (клики по кнопкам, ввод текста в поля).

## Архитектурные решения и фрагменты кода

### Организация SPA и роутинг

Приложение является одностраничным (Single Page Application). Переход между каталогом услуг и детальной карточкой расчета происходит без перезагрузки страницы за счет очистки содержимого родительского контейнера и вызова метода `render()` нужной страницы.

```javascript
// Фрагмент перехода на детальную страницу (pages/main/index.js)
clickCard(e) {
    // Получаем уникальный идентификатор услуги из data-атрибута кнопки
    const cardId = e.target.dataset.id;

    // Создаем экземпляр новой страницы, передавая родительский контейнер и ID
    const servicePage = new ServicePage(this.parent, cardId);

    // Запускаем отрисовку новой страницы (старый контент стирается внутри render)
    servicePage.render();
}
