## Индивидуальная тематика и данные

Бэкенд-часть разработана для обслуживания данных интерфейса из предыдущей лабораторной работы по индивидуальной теме: **«Интерфейс баллистического центра по расчету космических траекторий перелета к Луне»**.

Структура данных `stocks.json` была переработана для хранения этапов орбитальных расчетов. Реализованы следующие предзаполненные этапы:
1. Опорная орбита (Расчет параметров выхода на низкую околоземную орбиту).
2. Отлетный импульс (Вычисление маневра TLI).
3. Коррекция курса (Планирование промежуточных маневров).
4. Орбита Луны (Расчет тормозного импульса).

## Теоретическая база применяемых технологий

При разработке серверной части были применены следующие концепции:
* **Express.js:** Гибкий веб-фреймворк, предоставляющий удобные абстракции над модулем `http` (маршрутизация, middleware), что позволяет фокусироваться на бизнес-логике.
* **Middleware (Промежуточное ПО):** Функции, имеющие доступ к объектам запроса (`req`) и ответа (`res`). Выполняются в строгом порядке подключения.
* **REST API:** Архитектурный стиль взаимодействия, где для операций над ресурсами используются стандартные HTTP-методы (`GET`, `POST`, `PATCH`, `DELETE`).

## Архитектурные решения и фрагменты кода

### Организация слоистой архитектуры

Приложение построено по принципу **Layered Architecture** для удобного масштабирования. Поток данных организован следующим образом:
1. **Entry Point (`index.js`):** Точка входа, где подключаются глобальные middleware (например, `express.json()` для парсинга тела запроса) и роутеры.
2. **Router (`routes/stocks.js`):** Определяет связь между HTTP-методами/URL и соответствующими методами контроллера.
3. **Controller (`controllers/stocksController.js`):** Принимает запрос, извлекает параметры, вызывает бизнес-логику и управляет HTTP-статусами ответов (200, 201, 204, 400, 404).
4. **Service (`services/stocksService.js`):** Выполняет бизнес-логику (фильтрацию, генерацию новых ID, поиск).
5. **Data Layer (`services/fileService.js`):** Инкапсулирует логику чтения и записи в `stocks.json`.
